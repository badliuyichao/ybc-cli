## 快速开始（5 分钟上手）

### 1. 项目位置

```bash
cd D:\03-CODE\ybc-cli
```

### 2. 安装依赖

```bash
npm install
```

### 3. 构建 CLI

```bash
npm run build
```

### 4. 运行 CLI（查看帮助）

```bash
# 开发模式运行（使用 ts-node）
ts-node src/bin/ybc.ts --help

# 或者使用编译后的版本
node dist/bin/ybc.js --help
```

## 详细使用说明

### 一、项目结构概览

```
D:\03-CODE\ybc-cli\
├── src/                      # 源代码目录
│   ├── bin/ybc.ts           # CLI 入口文件
│   ├── cli/                 # CLI 命令实现
│   │   ├── commands/        # 所有命令
│   │   │   ├── config/      # 配置管理命令
│   │   │   └── generated/   # 自动生成的业务命令
│   │   └── output/          # 输出格式化器

├── dist/                    # 编译后的代码（npm run build 生成）
├── tests/                   # 测试文件
├── docs/                    # 文档
├── openapi/                 # OpenAPI 规范
├── package.json             # 项目配置
└── tsconfig.json            # TypeScript 配置
```

### 二、安装和构建

#### 2.1 安装依赖

```bash
# 安装所有依赖包
npm install

# 验证安装成功
npm list --depth=0
```

**依赖包说明**：
- `commander`: CLI 框架
- `axios`: HTTP 客户端
- `cli-table3`: 表格输出
- `chalk`: 颜色输出
- `typescript`: TypeScript 编译器
- `jest`: 测试框架
- `ts-node`: 开发模式运行 TypeScript

---

#### 2.2 构建 CLI

```bash
# 编译 TypeScript 到 JavaScript
npm run build

# 验证构建成功
ls dist/bin/ybc.js
```

**构建输出**：`dist/` 目录（包含所有编译后的 JavaScript 文件）

---

#### 2.3 生成 API 客户端和命令

如果需要重新生成（或修改了 OpenAPI 规范）：

```bash
# 从 OpenAPI 规范生成 API 客户端
npm run generate:api

# 从 API 客户端生成 CLI 命令
npm run generate:commands

# 重新构建
npm run build
```

### 三、运行 CLI 命令

#### 3.1 开发模式（推荐调试使用）

**使用 ts-node 直接运行 TypeScript 源码**：

```bash
# 查看帮助
ts-node src/bin/ybc.ts --help

# 查看版本
ts-node src/bin/ybc.ts --version

# 配置初始化（交互式）
ts-node src/bin/ybc.ts config init

# 查看当前配置
ts-node src/bin/ybc.ts config show

# 修改配置
ts-node src/bin/ybc.ts config set ak YOUR_AK
ts-node src/bin/ybc.ts config set sk YOUR_SK

# 执行业务命令（使用 Mock API）
ts-node src/bin/ybc.ts staff query --code EMP001 --format json
ts-node src/bin/ybc.ts todo list --status pending --format table
```

---

#### 3.2 编译模式（推荐生产使用）

**使用编译后的 JavaScript**：

```bash
# 查看帮助
node dist/bin/ybc.js --help

# 配置管理
node dist/bin/ybc.js config init
node dist/bin/ybc.js config show

# 业务命令
node dist/bin/ybc.js staff query --code EMP001 --format json
```

**性能对比**：
- 开发模式（ts-node）：启动时间 ~1000ms（需编译 TypeScript）
- 编译模式（node）：启动时间 ~100-200ms（直接运行 JS）

#### 3.3 全局安装（可选）

如果您想在任何目录使用 `ybc` 命令：

### 四、配置管理

#### 4.1 配置文件位置

**配置文件路径**：`~/.ybc/config.json`

- **Windows**: `C:\Users\<您的用户名>\.ybc\config.json`
- **macOS/Linux**: `~/.ybc/config.json`

**Token 缓存文件**：`~/.ybc/token.json`

---

#### 4.2 配置初始化

```bash
# 交互式配置（推荐）
ts-node src/bin/ybc.ts config init

# 手动配置
ts-node src/bin/ybc.ts config set ak YOUR_ACCESS_KEY
ts-node src/bin/ybc.ts config set sk YOUR_SECRET_KEY
ts-node src/bin/ybc.ts config set env sandbox  # sandbox 或 production
```

**配置内容**：
```json
{
  "ak": "YOUR_ACCESS_KEY",
  "sk": "ENCRYPTED_SECRET_KEY",  // 加密存储
  "env": "sandbox",
  "format": "table"
}
```

---

#### 4.3 查看配置

```bash
ts-node src/bin/ybc.ts config show
```

**输出示例**：
```
当前配置:
  AK: YOUR_ACCESS_KEY
  SK: YOUR****（已加密存储）
  环境: sandbox
  默认格式: table
```

**注意**：SK 会脱敏显示（只显示前 4 位 + ****）

---

#### 4.4 环境变量配置（CI/CD 推荐）

如果不想使用配置文件，可以通过环境变量：

```bash
# 设置环境变量
export YBC_AK="YOUR_ACCESS_KEY"      # Windows: set YBC_AK=YOUR_ACCESS_KEY
export YBC_SK="YOUR_SECRET_KEY"
export YBC_FORMAT="json"
export YBC_ENV="sandbox"

# 直接执行命令（无需 config init）
ts-node src/bin/ybc.ts staff query --code EMP001
```

**优先级**：环境变量 > 配置文件

### 五、业务命令使用

#### 5.1 可用命令列表

**已实现的命令**（Phase 1）：

| 命令 | 说明 | 示例 |
|------|------|------|
| `config init` | 配置初始化 | `ybc config init` |
| `config show` | 查看配置 | `ybc config show` |
| `config set <key> <value>` | 修改配置 | `ybc config set ak NEW_AK` |
| `staff query` | 查询人员信息 | `ybc staff query --code EMP001` |
| `staff enable <id>` | 启用员工 | `ybc staff enable STAFF001` |
| `staff disable <id>` | 禁用员工 | `ybc staff disable STAFF001` |
| `todo list` | 查询待办列表 | `ybc todo list --status pending` |
| `todo create` | 创建待办 | `ybc todo create --title "任务"` |

---

#### 5.2 命令选项

**全局选项**（所有命令可用）：

```bash
--format <json|table|csv|raw>  # 输出格式（默认 table）
--raw                          # 仅输出原始 JSON
--verbose                      # 详细日志（调试用）
--no-color                     # 禁用颜色
--dry-run                      # 预览请求（不发送）
--help, -h                     # 查看命令帮助
--version, -V                  # 查看版本
```

**示例**：

```bash
# JSON 格式输出（机器友好）
ts-node src/bin/ybc.ts staff query --code EMP001 --format json

# 表格格式输出（人类友好）
ts-node src/bin/ybc.ts staff query --code EMP001 --format table

# CSV 格式输出（Excel 导出）
ts-node src/bin/ybc.ts staff query --format csv > staff.csv

# 原始 JSON（无格式化）
ts-node src/bin/ybc.ts staff query --raw

# 详细日志（调试）
ts-node src/bin/ybc.ts staff query --verbose

# 预览请求（不发送）
ts-node src/bin/ybc.ts staff query --dry-run
```

#### 5.3 业务命令示例

**人员管理（staff）**：

```bash
# 查询人员信息

  --dept-id D001 \
```

**待办管理（todo）**：

```bash
# 查询待办列表

  --status pending \
  --assignee user001 \
  --format table

# 创建待办
ts-node src/bin/ybc.ts todo create \
  --title "完成任务" \
  --assignee user001 \
  --priority high
```

### 六、输出格式说明

#### 6.1 JSON 格式（机器友好）

```bash
ts-node src/bin/ybc.ts staff query --format json
```

**输出示例**：
```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "id": "EMP001",
      "name": "张三",
      "dept": "研发部",
      "status": "在职"
    }

  "pagination": {
    "page": 1,
    "size": 20,
    "total": 100
```

**特点**：
- 稳定的键顺序（便于 diff 和比较）
- 美化缩进（2 空格）

```bash
ts-node src/bin/ybc.ts staff query --format table
```

**特点**：
- Excel 可直接打开

### 八、常见问题与解决方案

#### 8.1 配置问题

**问题：运行命令提示 "配置不存在"**

```bash
# 解决方案：初始化配置
ts-node src/bin/ybc.ts config init
```

---

**问题：SK 输入错误，无法获取 Token**

```bash
# 解决方案：重新配置 SK
ts-node src/bin/ybc.ts config set sk YOUR_CORRECT_SK

# 或使用环境变量
export YBC_SK="YOUR_CORRECT_SK"
```

---

#### 8.2 Token 问题

**问题：Token 过期**

```bash
# ybc 会自动刷新 Token，无需手动处理

# 如果自动刷新失败，手动清除缓存
rm ~/.ybc/token.json  # Windows: del %USERPROFILE%\.ybc\token.json

# 重新执行命令（会自动获取新 Token）
ts-node src/bin/ybc.ts staff query --code EMP001
```

---

**问题：401 错误（鉴权失败）**

```bash
# 检查 AK/SK 是否正确
ts-node src/bin/ybc.ts config show

# 重新配置
ts-node src/bin/ybc.ts config init
```

---

#### 8.3 网络问题

**问题：网络连接超时**

```bash
# 检查网络连接
ping openapi-sit.yonyoucloud.com

# 使用 verbose 查看详情
ts-node src/bin/ybc.ts staff query --verbose

# 错误码：5（网络错误）
# 退出码含义：
#   0 - 成功
#   1 - 通用错误
#   4 - 业务错误（API 返回错误）
#   5 - 网络错误
#   6 - 鉴权错误
```

---

#### 8.4 性能问题

**问题：CLI 启动慢（开发模式）**

```bash
# 开发模式（ts-node）启动时间：~1000ms
# 这是正常的，因为需要编译 TypeScript

# 使用编译版本（更快）
npm run build
node dist/bin/ybc.js staff query

# 编译版本启动时间：~100-200ms
```

---

#### 8.5 权限问题

**问题：配置文件权限错误**

```bash
# ybc 自动设置文件权限为 600（仅当前用户可读写）

# 如果手动修改了权限，恢复：
chmod 600 ~/.ybc/config.json     # macOS/Linux
# Windows: 文件属性 → 安全 → 仅当前用户
```

#### 10.3 CI/CD 集成

**GitHub Actions 示例**：

```yaml
name: Sync Staff Data

on:
  schedule:
    - cron: '0 9 * * *'  # 每天 9 点

jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - uses: actions/setup-node@v2
        with:
          node-version: '16'

      - name: Install ybc
        run: npm install -g ybc

      - name: Configure
        env:
          YBC_AK: ${{ secrets.YBC_AK }}
          YBC_SK: ${{ secrets.YBC_SK }}
        run: |
          ybc config set ak $YBC_AK
          ybc config set sk $YBC_SK

      - name: Query Staff
        run: ybc staff query --format csv > staff.csv

      - name: Upload CSV
        uses: actions/upload-artifact@v2
        with:
          name: staff-data
          path: staff.csv
```
