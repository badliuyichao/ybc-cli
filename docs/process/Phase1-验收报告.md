## 3. 性能测量数据

### 3.1 性能基准测试结果

| 指标 | 实测值 | 目标值 | 达标 | 备注 |
|------|--------|--------|------|------|
| **CLI 启动时间** | **1038ms** | <500ms | ❌ 未达标 | ts-node 启动开销较大 |
| **首次 API 调用时间** | **1057ms** | <2000ms | ✅ 达标 | 包含 token 获取 |
| **后续 API 调用时间** | **1045ms** | <1000ms | ❌ 未达标 | ts-node 启动开销 |

**⚠️ 性能瓶颈分析**：

- **主要问题**：CLI 使用 `ts-node` 运行，启动开销约 1000ms
- **根本原因**：开发模式未编译，每次运行需要 TypeScript 编译
- **影响范围**：仅影响开发调试，不影响生产使用（编译后性能可优化）
- **优化建议**：
  1. Phase 2 使用 `npm run build` 编译后的代码（`dist/bin/ybc.js`）
  2. 添加 `--cache` 选项缓存 TypeScript 编译结果
  3. 使用 esbuild 或 swc 替代 tsc 加速编译
