### 1.2 核心组件说明

| 层级 | 组件 | 职责 |
|------|------|------|
| **CLI 层** | commander | 命令行解析、子命令注册、参数验证 |
| | 帮助系统 | 三级帮助、命令搜索、统计信息 |
| | 输出适配器 | JSON/Table/CSV/RAW 格式化输出 |
| **Service 层** | 配置管理器 | AK/SK 配置、环境管理、配置文件读写 |
| | 鉴权管理器 | Token 获取、刷新、缓存管理 |
| | 命令路由器 | 根据 OpenAPI 规范自动注册业务命令 |
| | Token 缓存 | 内存缓存、文件持久化、过期检测 |
| | 错误处理器 | 统一错误处理、退出码管理、用户友好提示 |
| | 日志管理器 | 日志级别控制、格式化输出 |
| **API 层** | OpenAPI Generated Client | 根据 OpenAPI 规范自动生成的 TypeScript 客户端 |
| | Business API Wrappers | 各业务域 API 封装（Staff/Todo/Voucher 等） |
| **Infrastructure 层** | 配置存储 | `~/.ybc/config.json` 配置文件管理 |
| | Token 存储 | `~/.ybc/token.json` Token 持久化 |
| | 加密服务 | SK 加密、解密（可选 keytar） |
| | HTTP 客户端 | axios 实例、请求/响应拦截器 |

### 2.3 Token 自动刷新机制

```
┌────────────────────────────────────────────────────────┐
│               Token 生命周期管理                       │
└────────────────────────────────────────────────────────┘

初始状态：
  Token 不存在
      ↓
  调用 /token API 获取新 Token
      ↓
  保存到 ~/.ybc/token.json
      ↓
  设置 expires_at = now + expires_in - 300s (提前5分钟过期)

每次调用前检查：
  if now >= expires_at:
      自动刷新 Token
  else if now >= expires_at - 300s:  (提前5分钟续期)
      后台异步刷新 Token

错误处理：
  401 错误 → 清除缓存 Token
         → 重新获取
         → 重试原请求（最多重试1次）
         → 失败则退出码 6

安全措施：
  - SK 不写入日志
  - Token 文件权限 600
  - 环境变量优先级高于配置文件
```

## 5. 技术选型详细说明

### 5.1 核心技术栈

| 技术 | 选型 | 理由 | 优势 | 劣势 |
|------|------|------|------|------|
| **运行语言** | TypeScript | 类型安全、IDE 友好、Node.js 生态 | 编译时类型检查、重构友好、文档自动生成 | 需要编译步骤、学习曲线略高 |
| **CLI 框架** | commander.js | 成熟稳定、社区活跃、功能完整 | 子命令支持、自动 help、参数解析、npm 生态 | 相比 oclif 功能较少，但足够用 |
| **HTTP 客户端** | axios | OpenAPI 生成器默认、拦截器强大 | 请求/响应拦截、自动转换、取消请求、广泛使用 | 包体积较大 |
| **配置管理** | conf | 简单易用、支持加密 | 自动路径、类型安全、加密字段 | 功能简单，不满足复杂场景时可自实现 |
| **表格输出** | cli-table3 | 功能丰富、支持颜色 | 自动列宽、样式定制、Unicode 支持 | 性能一般，大数据量需优化 |
| **加密** | crypto-js + keytar (可选) | 灵活、跨平台 | crypto-js 纯 JS 实现，keytar 系统级安全 | keytar 需编译原生模块 |
| **代码生成** | @openapitools/openapi-generator-cli | 自动化、类型安全 | 保持与 API 文档同步、减少手工维护 | 依赖 OpenAPI 规范质量 |
| **日志** | winston 或自实现 | 灵活、可扩展 | 多传输、日志级别、格式化 | winston 较重，自实现更轻量 |

### 5.2 开发工具链

| 工具 | 选型 | 用途 |
|------|------|------|
| **包管理** | npm | 依赖管理、发布 |
| **构建工具** | tsc + ts-node | TypeScript 编译、开发运行 |

| **CI/CD** | GitHub Actions | 自动化测试、发布 |

### 5.3 关键技术决策

#### 5.3.1 为什么选择 OpenAPI 代码生成？

**问题**：用友 BIP 有 300+ API，手工封装工作量巨大，且难以保持同步。

**方案对比**：

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| 手工封装每个 API | 灵活、可控 | 工作量大、易出错、难维护 | ❌ 不采用 |
| 动态代理（类似 Proxy） | 无需预定义 | 无类型提示、IDE 不友好 | ❌ 不采用 |
| OpenAPI 代码生成 | 自动化、类型安全、同步更新 | 依赖规范质量 | ✅ 采用 |

**实施策略**：
1. 维护一个 OpenAPI 规范文件（YAML/JSON）
2. 使用 `openapi-generator-cli` 生成 TypeScript-axios 客户端
3. 构建时自动生成，保证与规范同步
4. 只封装必要的业务逻辑，底层 API 调用交给生成代码

#### 5.3.2 为什么选择 commander.js 而非 oclif？

**对比**：

| 维度 | commander.js | oclif |
|------|--------------|-------|
| 学习曲线 | 低 | 中高 |
| 功能 | 满足需求 | 非常强大 |
| 包体积 | 小 | 大 |
| 插件系统 | 无 | 有 |
| 社区活跃度 | 高 | 中 |
| 自动化程度 | 中（需手工编写） | 高（脚手架自动生成） |

**决策**：选择 commander.js
- 理由：项目规模中等，commander.js 功能足够，且轻量、简单
- oclif 更适合复杂 CLI 工具（如多插件架构）

#### 5.3.3 Token 缓存策略

**方案**：
- **存储位置**：`~/.ybc/token.json`
- **存储格式**：`{ access_token, expires_at, token_type }`
- **文件权限**：600（仅用户可读写）
- **过期策略**：提前 5 分钟续期（避免临界点失效）
- **刷新策略**：
  - 同步刷新：当前请求等待刷新完成
  - 后台刷新：在过期前 5 分钟，下次请求前异步刷新

**替代方案对比**：

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| 内存缓存 | 速度最快 | 进程退出即丢失 | ❌ 不适合 CLI |
| 文件缓存 | 持久化、简单 | 需处理并发、权限 | ✅ 采用 |
| 数据库 | 功能强大 | 过度设计、依赖重 | ❌ 不适合 |

#### 5.3.4 SK 加密方案

**需求**：SK 不得明文存储在配置文件中。

**方案**：

```
优先级 1: 系统密钥链（推荐，最安全）
  └─ 使用 keytar 访问 macOS Keychain / Windows Credential Manager / Linux Secret Service
  └─ SK 加密存储在系统级密钥链中
  └─ 配置文件只存储 AK

优先级 2: 加密文件存储
  └─ 使用 AES-256-GCM 加密 SK
  └─ 密钥派生自用户密码或机器特征
  └─ 配置文件存储加密后的 SK

优先级 3: 环境变量（CI 场景）
  └─ YBC_AK / YBC_SK 环境变量
  └─ 不写入任何文件
```

**实施**：
- 开发阶段：优先实现环境变量方案（最简单）
- MVP 阶段：实现加密文件存储
- 正式版本：集成系统密钥链（keytar）

## 7. 性能与可靠性设计

### 7.1 性能优化策略

| 场景 | 目标 | 策略 |
|------|------|------|
| CLI 启动速度 | < 500ms | 延迟加载非核心模块、优化 import 顺序 |
| 首次 API 调用 | < 2s（含 Token 获取） | Token 缓存、并行初始化 |
| 后续 API 调用 | < 1s（缓存 Token） | HTTP Keep-Alive、连接池 |
| 帮助命令 | < 100ms | 预生成帮助文本、懒加载命令列表 |
| 大数据量输出 | < 3s（1000 行） | 分页、流式输出、限制最大行数 |

### 7.2 可靠性保障

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Token 过期 | API 调用失败 | 自动刷新、提前续期、重试机制 |
| 网络不稳定 | 请求超时 | 超时配置、重试策略、错误提示 |
| API 限流 | 请求被拒绝 | 速率限制提示、指数退避重试 |
| 配置文件损坏 | 无法启动 | 配置校验、备份机制、恢复建议 |
| OpenAPI 规范变更 | 生成代码失效 | 版本锁定、变更检测、兼容性检查 |

### 7.3 错误处理策略

**分层错误处理**：

```
CLI 层错误
  ├─ 命令解析错误 → 退出码 1，显示帮助
  ├─ 参数验证错误 → 退出码 1，显示错误提示
  └─ 用户中断 → 退出码 130

Service 层错误
  ├─ 配置错误 → 退出码 1，提示重新配置
  ├─ Token 获取失败 → 退出码 6，提示检查凭证
  └─ 权限不足 → 退出码 6，提示检查权限

API 层错误
  ├─ 401 Unauthorized → 自动刷新 Token 并重试（最多 1 次）
  ├─ 429 Too Many Requests → 退出码 4，提示限流
  ├─ 5xx Server Error → 退出码 5，提示服务端问题
  └─ 网络错误 → 退出码 5，提示网络问题

业务错误
  └─ API 返回 code ≠ 0 → 退出码 4，显示业务错误信息
```

**用户友好提示**：
- 不打印堆栈跟踪（除非 `--verbose`）
- 提供明确的错误原因和建议操作
- 包含错误码供查询文档

## 9. 安全设计

### 9.1 安全威胁模型

| 威胁 | 风险 | 缓解措施 |
|------|------|----------|
| SK 泄露 | 凭证被盗用，数据泄露 | SK 加密存储、环境变量注入、不明文日志 |
| Token 泄露 | 短期风险，会话劫持 | Token 文件权限 600、短期有效（2 小时） |
| 中间人攻击 | 请求被监听 | HTTPS 强制、证书校验 |
| 命令注入 | 恶意参数执行 | 参数严格校验、禁用 eval |
| 配置篡改 | 凭证被修改 | 配置文件权限控制、校验签名 |

### 9.2 安全实施清单

- [ ] SK 不出现在日志、错误信息、命令行历史中
- [ ] 配置文件权限设置为 600（仅用户可读写）
- [ ] Token 文件权限设置为 600
- [ ] 支持 SK 加密存储（可选 keytar）
- [ ] 环境变量优先级高于配置文件（支持 CI 场景）
- [ ] HTTPS 强制，禁用不安全 HTTP
- [ ] 证书校验，禁用 `rejectUnauthorized: false`
- [ ] 参数严格校验，防止注入攻击
- [ ] 不执行动态代码（`eval`、`Function`）
- [ ] 敏感信息脱敏显示（`config show` 只显示 SK 后 4 位）
- [ ] 提供 `ybc config clear` 清除所有敏感信息

### 9.3 安全审计机制

**操作日志**：

  result: 'success' | 'failure';

### ADR-001: 选择 TypeScript 而非 JavaScript

**背景**：CLI 工具需要类型安全，且 Node.js 生态支持良好。

**决策**：使用 TypeScript。

**理由**：
- 编译时类型检查，减少运行时错误
- IDE 友好，重构安全
- 接口定义清晰，文档自动生成
- 团队熟悉度

**后果**：
- 需要编译步骤
- 开发环境配置略复杂

---

### ADR-002: 使用文件缓存而非数据库

**背景**：需要持久化 Token 和配置。

**决策**：使用 JSON 文件存储。

**理由**：
- CLI 工具数据量小，无需数据库
- 文件简单、无依赖
- 易于调试和手动修改
- 跨平台兼容性好

**后果**：
- 并发写入需要加锁
- 需要处理文件权限

---

### ADR-003: OpenAPI 代码生成而非手工封装

**背景**：300+ API，手工封装工作量大。

**决策**：使用 OpenAPI Generator 自动生成客户端。

**理由**：
- 自动化程度高
- 保持与 API 文档同步
- 类型安全
- 减少维护成本

**后果**：
- 依赖 OpenAPI 规范质量
- 需要维护规范文件
- 生成代码可能冗余
