# 获取租户所在数据中心域名

## 说明

由于 YonBIP 升级到多数据中心版，开放平台也同步升级支持多数据中心。为了以后分流，开放平台是每个数据中心部署独立一套，所以开发者调用开放平台接口，就需要先获取到租户所在数据中心的核心网关域名和 auth 域名，其中：

- **auth 域名**：用于调用「获取调用接口访问令牌 access_token」接口
- **核心网关域名**：用于调用业务接口

## 请求地址

| 项目 | 值 |
|------|-----|
| **URL** | `https://api.yonyoucloud.com/open-auth/dataCenter/getGatewayAddress?tenantId=xxxxxxxx` |
| **请求方法** | GET |

## 请求参数

| 字段 | 类型 | 说明 |
|------|------|------|
| tenantId | string | 租户ID |

## 返回参数说明

| 字段 | 类型 | 说明 |
|------|------|------|
| code | String | 结果码，正确返回 `"00000"` |
| message | String | 结果信息，若有错误，该字段会返回具体错误信息 |
| data.gatewayUrl | String | 租户所在数据中心的核心网关域名，调用业务接口时需要拼接核心网关域名和接口相对路径 |
| data.tokenUrl | String | 租户所在数据中心的 auth 域名，调用获取 token 接口时，需要拼接 auth 域名和获取 token 接口相对路径 |

## 返回数据示例

```json
{
  "code": "00000",
  "message": "成功！",
  "data": {
    "gatewayUrl": "https://yonbip.diwork.com/iuap-api-gateway",
    "tokenUrl": "https://yonbip.diwork.com/iuap-api-auth"
  }
}
```
