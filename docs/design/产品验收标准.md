---

## 1. 产品验收目标

确保 `ybc` CLI 工具满足以下核心目标:

- ✅ **一键安装使用**: 用户能通过 `npm install -g ybc` 快速安装并使用
- ✅ **零配置快速调用**: 一次配置 AK/SK 后，自动管理 token 生命周期
- ✅ **自然发现命令**: 三级帮助体系 + 关键词搜索，用户无需翻阅文档
- ✅ **双模输出**: 人类友好的表格与机器友好的 JSON，适配不同用户
- ✅ **安全可靠**: SK 本地加密/环境变量注入，绝不明文泄露

## 2. 用户故事（User Stories）

### 2.1 集成开发者角色

#### US-001: 快速配置并开始使用

**用户故事**:
> 作为一名集成开发者，我希望能够快速配置 AK/SK 并立即开始调用 API，以便高效对接 ERP 与第三方系统。

**验收条件**:
- [ ] `ybc config init` 命令能够交互式引导配置 AK/SK、环境
- [ ] 配置完成后，立即可以执行业务命令（如 `ybc staff query`）
- [ ] Token 自动获取并缓存，无需手动管理
- [ ] 配置文件权限自动设置为 600（安全）
- [ ] 配置错误时给出明确提示和修复建议

**优先级**: P0

---

#### US-002: 调试API调用细节

**用户故事**:
> 作为一名集成开发者，我希望能够查看完整的请求/响应细节，以便调试 API 调用问题。

**验收条件**:
- [ ] `--verbose` 选项输出详细请求信息（URL、Headers、Body）
- [ ] `--verbose` 选项输出详细响应信息（Status、Headers、Body）
- [ ] `--dry-run` 选项仅打印请求信息而不实际发送
- [ ] 详细日志格式清晰易读（JSON 格式）
- [ ] 错误时打印完整的错误堆栈（仅在 verbose 模式）

**优先级**: P1

---

#### US-003: 获取结构化输出用于脚本处理

**用户故事**:
> 作为一名集成开发者，我希望获得稳定的 JSON 输出格式，以便在脚本中进一步处理数据。

**验收条件**:
- [ ] `--format json` 选项输出纯 JSON 格式（无额外文本）
- [ ] JSON 格式稳定，键名统一、无多余字段
- [ ] JSON 输出可通过管道传递给其他工具（如 `jq`）
- [ ] 错误时也输出 JSON 格式的错误信息
- [ ] 支持 `--raw` 选项输出服务端原始响应（无任何转换）

**优先级**: P0

---

### 2.2 运维工程师角色

#### US-004: 非交互式批量调用

**用户故事**:
> 作为一名运维工程师，我希望通过环境变量注入凭证，并在脚本中批量调用多个命令，以便自动化运维任务。

**验收条件**:
- [ ] `YBC_AK` / `YBC_SK` 环境变量优先级高于配置文件
- [ ] `YBC_QUIET=1` 禁止所有进度、警告等非结果输出
- [ ] `YBC_FORMAT=json` 全局设置输出格式
- [ ] 退出码规范：成功 0、业务错误 4、网络错误 5、鉴权错误 6
- [ ] 支持在 shell 脚本中连续调用多个命令
- [ ] Token 自动管理，跨命令共享缓存

**优先级**: P0

---

#### US-005: 定时任务自动刷新Token

**用户故事**:
> 作为一名运维工程师，我希望在长时间运行的定时任务中 Token 自动续期，以便任务不中断。

**验收条件**:
- [ ] Token 过期前 5 分钟自动异步续期
- [ ] 401 错误时自动刷新 Token 并重试一次
- [ ] Token 刷新过程对用户透明（无日志输出，除非 verbose）
- [ ] 多个命令之间共享 Token 缓存
- [ ] Token 刷新失败时给出明确错误提示（退出码 6）

**优先级**: P1

---

#### US-006: 批量调用多个API

**用户故事**:
> 作为一名运维工程师，我希望从配置文件批量执行多个 API 调用，以便一次性同步多个数据。

**验收条件**:
- [ ] `ybc batch --file commands.json` 从文件批量执行命令
- [ ] 批量文件格式清晰（JSON数组，每个元素包含 domain、action、params）
- [ ] 执行结果按顺序输出（JSON格式，包含每个命令的结果）
- [ ] 单个命令失败不影响后续命令执行
- [ ] 批量执行完成后输出汇总报告（成功数、失败数）

**优先级**: P2（可选功能）

**用户故事**:

**验收条件**:
- [ ] 默认输出格式为表格（table）
- [ ] 表格自动调整列宽，适配终端宽度
- [ ] 表格支持颜色区分（表头、数据、状态）
- [ ] 表格支持 Unicode 字符（中文对齐）

**优先级**: P0

**用户故事**:

**验收条件**:
- [ ] `ybc search 待办` 搜索包含 "待办" 的命令
- [ ] 搜索结果列出完整命令名和简短描述
- [ ] 搜索支持中文、英文、部分拼音
- [ ] 搜索范围包含命令名、描述、参数名
- [ ] 搜索结果按相关性排序

**优先级**: P1

---

#### US-009: 查看帮助信息了解命令用法

**用户故事**:
> 作为一名业务顾问，我希望通过帮助信息快速了解命令用法，以便正确使用命令。

**验收条件**:
- [ ] `ybc help` 显示全局帮助（欢迎语、全局选项、命令分组）
- [ ] `ybc staff help` 显示该业务域下的所有命令
- [ ] `ybc staff query --help` 显示详细参数说明和示例
- [ ] 帮助信息包含参数说明、默认值、示例命令、API 文档链接
- [ ] 帮助信息格式清晰易读（颜色、分组）

**优先级**: P0

### 2.4 大模型 Agent 角色

#### US-010: 获取标准化JSON Schema

**用户故事**:
> 作为一名大模型 Agent，我希望获取命令的标准化 JSON Schema 描述，以便注册工具并正确调用。

**验收条件**:
- [ ] `ybc staff query --help-json` 输出 JSON Schema 格式
- [ ] JSON Schema 包含命令名、描述、参数定义（类型、描述）
- [ ] JSON Schema 符合 OpenAI Functions / Anthropic tool use 规范
- [ ] JSON Schema 稳定不变（命令更新时同步更新 Schema）
- [ ] Schema 输出格式化（易读）

**优先级**: P1

---

#### US-011: 稳定的退出码和错误信息

**用户故事**:
> 作为一名大模型 Agent，我希望根据退出码判断执行状态，以便自动化处理不同类型的错误。

**验收条件**:
- [ ] 成功: 退出码 0
- [ ] 业务错误: 退出码 4（API返回 code≠0）
- [ ] 网络错误: 退出码 5（连接超时、DNS失败）
- [ ] 鉴权错误: 退出码 6（AK/SK错误）
- [ ] 错误信息以 JSON 格式输出（便于解析）
- [ ] 错误信息包含错误类型、错误描述、建议操作

**优先级**: P0

---

#### US-012: 无交互模式自动执行

**用户故事**:
> 作为一名大模型 Agent，我希望在无交互模式下自动执行命令，以便避免用户交互阻塞。

**验收条件**:
- [ ] 所有命令支持非交互模式（无 `prompt` 等）
- [ ] 必需参数缺失时直接报错（退出码 1），不进入交互
- [ ] 环境变量 `YBC_INTERACTIVE=never` 禁用交互模式
- [ ] 输出固定格式（JSON），无颜色、无进度条
- [ ] 错误时立即退出，无恢复询问

**优先级**: P0

### 4.3 输出格式验收标准

```json
{
  "code": 0,
  "message": "success",

      "dept": "研发部",

- [ ] 数据类型正确（字符串、数字、布尔）
- [ ] 错误时也输出JSON格式

```
┌─────────┬────────┬──────────┬────────┐
│ code    │ name   │ dept     │ status │
├─────────┼────────┼──────────┼────────┤

验收标准:
- [ ] 表头颜色区分（蓝色）

- [ ] 中文对齐正确（考虑Unicode宽度）

## 11. 缺陷优先级定义

| 严重程度 | 定义 | 示例 | 处理时限 |
|----------|------|------|----------|
| **P0 - 致命** | 功能完全无法使用，影响核心流程 | config init失败、Token获取失败、所有API调用失败 | 立即修复，阻塞发布 |
| **P1 - 严重** | 主要功能受损，但有替代方案 | 输出格式错误、搜索失败、帮助信息缺失 | 24小时内修复，不阻塞发布 |
| **P2 - 一般** | 次要功能问题，不影响核心流程 | CSV格式化小问题、性能略慢、提示不够友好 | 1周内修复 |
| **P3 - 轻微** | 界面细节、文案优化 | 颜色不够美观、提示文案优化 | 下版本修复 |
