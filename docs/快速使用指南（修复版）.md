# ybc CLI 快速使用指南（修复版）

---

## 🚀 快速开始（3 步上手）

### 步骤 1：进入项目目录
```bash
cd D:\03-CODE\ybc-cli
```

### 步骤 2：安装依赖
```bash
npm install
```

### 步骤 3：运行 CLI（选择一种方式）

#### ✅ 方式 A：使用 npx（最简单，推荐）
```bash
npx ts-node src/bin/ybc.ts --help
npx ts-node src/bin/ybc.ts config init
npx ts-node src/bin/ybc.ts staff query --code EMP001 --format json
```

#### ✅ 方式 B：使用 npm run dev（更规范）
```bash
npm run dev -- --help
npm run dev -- config init
npm run dev -- staff query --code EMP001 --format json
```

#### ✅ 方式 C：使用编译版本（最快）
```bash
# 先构建
npm run build

# 使用编译版本（启动快 5-10 倍）
node dist/bin/ybc.js --help
node dist/bin/ybc.js config init
node dist/bin/ybc.js staff query --code EMP001 --format json
```

---

## 💡 为什么会报错？

**错误信息**：`ts-node: The term 'ts-node' is not recognized...`

**原因**：`ts-node` 已安装在项目中，但不在全局路径中，所以不能直接运行 `ts-node` 命令。

**解决**：使用 `npx ts-node` 或 `npm run dev` 来调用。

---

## 📊 三种方式对比

| 方式 | 命令 | 启动时间 | 适用场景 |
|------|------|---------|---------|
| **npx ts-node** | `npx ts-node src/bin/ybc.ts` | ~1000ms | 开发调试（最简单） |
| **npm run dev** | `npm run dev --` | ~1000ms | 开发调试（更规范） |
| **node dist** | `node dist/bin/ybc.js` | ~100-200ms ⚡ | 生产使用（最快） |

## 🎯 基本使用流程

### 1. 配置初始化

```bash
# 方式 A（推荐）
npx ts-node src/bin/ybc.ts config init

# 方式 B
npm run dev -- config init

# 方式 C（编译版本）
npm run build
node dist/bin/ybc.js config init
```

**交互式配置**：
- 输入 AK（Access Key）
- 输入 SK（Secret Key）
- 选择环境（sandbox 或 production）

---

### 2. 查看配置

```bash
npx ts-node src/bin/ybc.ts config show
```

**输出示例**：
```
当前配置:
  AK: YOUR_ACCESS_KEY
  SK: YOUR****（已加密存储）
  环境: sandbox
```

---

### 3. 执行业务命令

```bash
# 查询人员信息（JSON 格式）
npx ts-node src/bin/ybc.ts staff query --code EMP001 --format json

# 查询待办列表（表格格式）
npx ts-node src/bin/ybc.ts todo list --format table

# 导出 CSV
npx ts-node src/bin/ybc.ts staff query --format csv > staff.csv
```

## 🎨 输出格式选项

```bash
--format json   # JSON 格式（机器友好，适合 AI agent）
--format table  # 表格格式（人类友好，默认）
--format csv    # CSV 格式（Excel 导出）
--raw           # 原始 JSON（管道传输）
--verbose       # 详细日志（调试用）
--dry-run       # 预览请求（不发送）
--no-color      # 禁用颜色（CI/CD）
```

**示例**：

```bash
# JSON 输出
npx ts-node src/bin/ybc.ts staff query --format json

# 表格输出
npx ts-node src/bin/ybc.ts staff query --format table

# CSV 输出（导出 Excel）
npx ts-node src/bin/ybc.ts staff query --format csv > staff.csv

# 详细日志（调试）
npx ts-node src/bin/ybc.ts staff query --verbose

# 预览请求（不发送）
npx ts-node src/bin/ybc.ts staff query --dry-run
```

## ❓ 常见问题

### Q1: ts-node 报错怎么办？

**答**：使用 `npx ts-node` 或 `npm run dev`，不要直接运行 `ts-node`。

---

### Q2: 哪种方式最推荐？

**答**：
- **开发调试**：使用 `npx ts-node src/bin/ybc.ts`（最简单）
- **生产使用**：使用 `npm run build && node dist/bin/ybc.js`（最快）

---

### Q3: npm run dev 后面为什么需要加 `--`？

**答**：`--` 用于分隔 npm 命令和传递给脚本的参数。

**示例**：
```bash
# 错误（参数不会传递给 ybc）
npm run dev --help

# 正确（help 参数传递给 ybc）
npm run dev -- --help
```

---

### Q4: 如何全局安装（可选）？

**答**：如果想在任何目录使用 `ybc` 命令：

```bash
# 全局安装
npm install -g .

# 现在可以在任何目录使用
ybc --help
ybc config init

# 全局卸载
npm uninstall -g ybc
```
